O que é o Cisco Packet Tracer?

O Cisco Packet Tracer é um simulador de rede desenvolvido pela Cisco para ensinar e aprender conceitos de redes de computadores. Ele permite que os usuários criem e configurem redes virtualmente, simulando dispositivos como roteadores, switches, computadores e dispositivos IoT, sem a necessidade de hardware físico.

Laboratório básico no Cisco Packet Tracer — preparar a topologia

  1. Abra o Packet Tracer e crie a seguinte topologia simples:
    • 1 Router (por exemplo, 1941)
    • 1 Switch (por exemplo, 2960)
    • 2 PCs (PC0 e PC1)
    • 1 Server (Server0)
  2. Interligue: Router — Switch — (PC0, PC1, Server0). Use cabos Copper Straight‑Through.
  3. No Server0 → Desktop → Services:
    • Ative o serviço HTTP (marque HTTP) e, opcionalmente, coloque uma página index.html simples.
  4. No PC0 e PC1 → Desktop → IP Configuration: atribua IPs estáticos na mesma rede do router. Exemplo:
    • PC0: 192.168.10.10 / 255.255.255.0, Gateway 192.168.10.1
    • Server0: 192.168.10.20 / 255.255.255.0, Gateway 192.168.10.1
    • Router (interface conectada ao switch): 192.168.10.1 / 255.255.255.0

Configuração básica do Router (CLI)

Se preferires configurar o router por CLI, exemplo mínimo (entra no modo de configuração):


    enable
    configure terminal
    interface GigabitEthernet0/0
     ip address 192.168.10.1 255.255.255.0
     no shutdown
    exit
    interface GigabitEthernet0/1
     no shutdown
    exit
    end
    write memory

Nota: adapta nomes de interfaces e endereços conforme a tua topologia.

Gerar tráfego HTTP para inspeção

  1. No PC0 → Desktop → Web Browser: na barra de endereços coloca http://192.168.10.20 (IP do Server0) e carrega para gerar uma requisição HTTP.
  2. Se o Server tiver uma página, o browser mostrará o conteúdo; isto gera os pacotes que vamos inspecionar.

Usar o Modo Simulation para inspecionar pacotes

  1. Muda para o modo Simulation (canto inferior direito — botão "Real Time" → "Simulation").
  2. Na área "Packet Filters" selecciona apenas os protocolos que queres ver (por exemplo, marque apenas HTTP para filtrar tráfego da aplicação web).
  3. Carrega em "Auto Capture / Play" (play) para que o Packet Tracer capture e avance automaticamente os eventos.
  4. Quando um evento aparecer na timeline, clica sobre ele e escolhe Inspect (ou clica duas vezes na linha do evento). Vais ver:
    • Resumo do pacote (origem, destino, protocolo).
    • Detalhes por camadas: Ethernet, IP, TCP e HTTP (com cabeçalhos e, se aplicável, payload).
  5. Explora campos como Source IP, Destination IP, portas TCP, flags (SYN, ACK) e o corpo HTTP (GET/POST).

Exemplo básico de uso / exercícios sugeridos

  • Inspecionar um GET: abre uma página no Server0 e analisa no Simulation como é construída a requisição GET e a resposta 200 OK.
  • Bloquear com ACL: adiciona uma ACL no router para bloquear PC1 de aceder ao Server0 e observa a diferença no Simulation.
  • Alterar filtros: remove o filtro HTTP para ver tudo (ARP, ICMP, TCP) e observar a sequência completa (ARP request/reply, TCP handshake, HTTP).
  • Gerar tráfego ICMP: usa o Terminal do PC (Desktop → Command Prompt) com ping 192.168.10.20 e observa os ICMP packets.

Dicas e boas práticas

  • Usa o Simulation para estudar cada passo do pacote — é uma ferramenta educativa excelente para compreender protocolos.
  • Salva frequentemente o projeto (.pkt) para não perder a topologia.
  • O Packet Tracer não substitui ferramentas reais de captura como tcpdump ou Wireshark, mas é ótimo para aprender conceitos sem riscos.
  • Faz testes apenas em ambientes controlados e com autorização — nunca em redes alheias.
